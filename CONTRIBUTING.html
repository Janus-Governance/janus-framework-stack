Contributing to Janus Governance Core

Janus Governance Core is a specification repository. Changes to core semantics must follow the RFC process.

Contribution types

Documentation improvements

Minor documentation improvements may be submitted directly as pull requests.

RFC proposals

Changes to the governance model, evidence semantics, or protocol definitions must be proposed through the RFC process.

Steps:

  1. Fork the repository
  2. Create a branch
  3. Add or update an RFC under /rfcs
  4. Open a Pull Request describing the proposal

Review model

RFC proposals are evaluated based on:

  • conceptual clarity
  • compatibility with Core Lite semantics
  • traceability and auditability
  • minimal governance complexity

Authority model

Final acceptance of RFC changes is determined by the project maintainers.
